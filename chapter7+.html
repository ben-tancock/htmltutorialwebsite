Chapter 7: Forms

The term 'forms' refers to different elements that allow you to collect information from visitors to your website.
The most well known example is probably Google's search bar.

Form Structure:

Form controls live inside the form tag. This element should always carry the action attribute and will usually have a method and id attribute too.

  • action: The value of the action attribute is the URL for the page on the server that will recieve the information in the form when it is submitted.
  • method: Forms can be sent using one of the two methods: get or post.
  • id: Value used to identify the form distinctly from other elements on the page (and is often used by scripts -- such as those that check you have entered the information into the fields that require values)
  • size: The size attribute should NOT be used on new forms.
  • input (tag that resides inside form): used to create several different form controls. The value of the type attribute determines what kind of input they will be creating.
    • type="text": When the type attribute has a value of text, it creates a single line of text input.
    • type="password": When the type attribute has the value of password, it creates a text box that acts just like a single line of text input, except the characters are blocked out.
    • name: When users enter the information into a form, the server needs to know which form control each piece of data was entered into. Therefore, each form control requires a name attribute.
    • maxlength: You can use the maxlength attribute to limit the number of characters a user may enter into the text field.
    • textarea: element is used to create multi-line text input. Unlike other input elements, this is not an empty element. It should therefore have an opening and closing tag.
