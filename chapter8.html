ID attributes:

id attributes in tags allow you to uniquely identify different tags and style them differently with CSS, even though they might be the same type of tag. I sort of already knew this and have done it before.

Some elements will always appear to start on a new line in the browser window. These are known as block elements. Other elements that appear to continue in the same line as their neighbouring elements are known as inline elements.

The div element allows you to group a set of elements together in one block-level box. Using an id or class attribute on a div element means you can style entire sections differently with CSS rules.

The span element acts like the inline equivelant of the div element. It is used to either:

  1. Contain a section of text where there is no other suitable element to differentiate it from its surrounding text
  2. Contain a number of inline elements
The most common reason why people use span elements is so that they can control the appearance of the content of these elements using CSS. You will usually see that a class or id attribute is used with span elements:
  • to explain the purpose of this span element
  • so that CSS styles can be applied to elements that have specific values for these attributes

IFrames:

An iframe is like a little window that has been cut into your page -- and in that window you can see another page. The term iframe is an abbreviation of inline frame. One commone use of iframes is to embed a google map into the page. An iframe is created with an iframe tag. There are a few attributes that you will need to know to use it:

  • src: specifies the URL of the page to show in the frame.
  • height: the height attribute specifies the height of the iframe in pixels.
  • width: you get the idea.
  • scrolling: won't be supported in HTML5, but it indicates whether the iframe should have scrollbars or not.
  • frameborder: also won't be supported in HTML5... it indicates whether the frame should have a border or not.
  • seamless: a new attribute in HTML5, can be applied to an iframe where scrollbars are not desired. Not supported in older browsers.

In google maps, you need to specifically click the 'embed in map' option to get the proper link for an iframe.

Other stuff:

Information about your pages:

  • meta tag:
    • description: Containts a description about the page. This is commonly used by search engines to understand what the page is about. Should be a max of 155 characters. Is sometimes is also displayed in search engine results.
    • keywords: This contains a list of comma-separated words that a user might search on to find the page. In practice, it no longer has any noticeable effect on how seach engines index your site.
    • robots: Indicates whether a search engine should add this page to their search results or not. A value of noindex can be useful if this page should not be added. A value of nofollow can be used if search engines should add this page in their results but not any pages that it links to.

The meta element also uses the http-equiv attributes in pairs. In our example, you can see three instances of the http-equiv attributes, each one has a different purpose:

  • author: This defines the author of the webpage
  • pragma: This prevents the browser from caching the page (storing it locally to save time downloading it on subsequent visits)
  • expires: Because browsers often cache content of a page, the expires option can be used to indicate when the page should expire (and no longer be cached). Note the date must be specified.

Escape Characters:

There are some characters that are used in and reserved by the HTML code. (For example the left and right angled brackets). Therefore, if you want these characters to appear on your page you need to use what are termed "escape" characters. For example, to write a left angled bracket, you can use either & + lt or & + #60 which = <. Here are them all listed in the book:

  • <
  • >
  • &
  • "
  • ¢
  • £
  • ¥
  • &euro
  • ©
  • ®
  • &trade
  • &lsquo
  • &rsquo
  • &ldquo
  • &rdquo
  • ×
  • ÷
