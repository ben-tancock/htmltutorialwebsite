Chapter 10: Introducing CSS

With CSS, we can customize elements by their block level (for blockline elements) or inline elements specifically, or both.

CSS works by associating rules with HTML elements. These rules govern how the content specified elements should be displayed. A CSS rule contains two parts: a selector and a declaration.

CSS selectors sit inside curly brackets and each is made up of two parts: a property and a value, separated by a semi-colon.

  • h1, h2, h3 { font-family: Arial; color: yellow;}
    • This rule indicates that all h1, h2, and h3 elements should be shown in the Arial typeface, in a yellow colour.

You can use both external and internal CSS. The difference is pretty simple: External is when you use the <link> tag (which resides in the head) to tell the browser where to find the CSS file used to style the page. Link is an empty element (meaning it does not need a closing tag) and has 3 attributes:

  • href:

    this specifies the path to the css file

  • type:

    this specifies the type of document being linked to

  • rel:

    this specifies the relationship between the html page and the file it is linked to. the value should be stylesheet when linking to a CSS file.

Internal CSS is when the CSS code is inside the HTML page code itself. It is declared using the <style> tag, which specifies the type attribute, and then the CSS code is contained within it.

When building a site with more than one page, it is expedient to use an external CSS style sheet. That's what we'll do from now on, and we'll generally avoid internal CSS styling.

CSS Selectors:

There are many different types of CSS selectors that allow you to target rules to specific elements in an HTML document. CSS selectors are case sensitive. So they must match element names and attribute values exactly. There are some more advanced selectors which will allow you to select elements based on attributes and their values. The table below lists the most commonly used CSS selectors.

SELECTOR MEANING EXAMPLE
UNIVERSAL SELECTOR Applies to all elements in the document * { }

targets all elements on the page
TYPE SELECTOR Matches element names h1, h2, h3 { }

Targets the h1, h2, and h3 elements
CLASS SELECTOR Matches an element whose class attribute has a value that matches the one specified after the period symbol .note { }

Targets any element whose class attribute has a value of note

p.note { }

Targets only <p> elements whose class attribute has a value of note

ID SELECTOR Matches an element whose id attribute has a value that matches the one specified after the pound or hash symbol #introduction { }

Targets the element whose id attribute has the value of introduction
CHILD SELECTOR Matches the element that is a direct child of another li>a { }

Targets any <a> elements that are children of an <li> element (but not other <a> elements in the page)
DESCENDANT SELECTOR Matches an element that is a descendant of another specified element (not just a direct child of that element) p a { }

Targets any <a> elements that sit inside a <p> element, even if there are other elements nested between them.
ADJACENT SIBLING SELECTOR Matches an element that is the next sibling of another h1+p { }

Targets the first <p> element after any <h1> element (but not other <p> elements)
GENERAL SIBLING SELECTOR Matches an element that is a sibling of another, although it does not have to be directly preceding an element h1~p { }

If you had two <p> elements that are siblings of an <h1> element, this rule would apply to both

How CSS Rules Cascade:

If there are two or more rules that apply to the same element, it is important to understand which take precedence.

  • LAST RULE: If the two selectors are identical, the latter of the two will take precedence.
  • SPECIFICITY: If one selector is more specific than the others, the more specific rule will take precedence over the more general ones. (e.g. h1 is more specific than *, p b is more specific than p, p#intro is more specific than p, etc.)
  • IMPORTANT: You can add !important after any property value to indicate that it should be considered more important than other rules that apply to the same element.

Inheritance:

If you specify the font-family or color properties on the <body> element, they will apply to most child elements. This is because the font-family property is inherited by child elements. It saves you from having to apply these properties to as many elements, and results in simpler style sheets.

Compare this with the background-color or border properties, which are NOT inherited by child elements. If these were inherited by all child elements then the page would look quite messy.

You can force a lot of properties to inherit values from their parent elements by using inherit for the value of the properties.
