CHAPTER 11 & 12 - COLOUR AND FONT:



You can specify colour in CSS in one of the three ways:

  • RBG Values:

    These express colours in terms of how much red, green, and blue are used to make it up.

  • Hex Codes:

    These are six-digit codes that represent the amount of red, green and blue in a colour. Preceeded by the hash symbol.

  • Color Names:

    There are 147 predefined colour names that are recognized by browsers.



Contrast:

When picking foreground and background colours, it is important to ensure that there is enough contrast for the text to be eligible.




LOW CONTRAST

MEDIUM CONTRAST

HIGH CONTRAST

Text is harder to read when there is low contrast between background and foreground colours.

A lack of contrast is particularly a problem for those with visual impairment or colour blindness.

It also affects those with poor monitors and sunlight on their screens.

Text is easier to read when there is higher contrast between the background and foreground colours. If you want people to read a lot of text on your page, however, then too much contrast can be harder to read too.

For long spans of text, reducing the contrast a little bit improves readability.

You can reduce the contrast by using dark grey text on a white background or an off-white text on a dark background.

CSS3: HSL COLOURS

CSS3 introduces an entirely new and intuitive way to specify colours using hue, saturation, and lightness values. These properties are modified with the hsl and hsla (hue, saturation, lightness, and alpha) attributes.



HUE SATURATION LIGHTNESS ALPHA

Hue is the colloquial idea for colour.

Saturation is the amount of grey in a coulour. Saturation is represented as a percentage. 100% is full saturation and 0% is a shade of grey.

Lightness is the amount of white or black in a colour. Like saturation, lightness is also represented as a percentage, where 100% is white, and 50% is normal.

Alpha represents transparency, and is expressed as a number from 0 and 1.0.

The formatting of your text can have a significant effect on how readable your pages are. As we look through these properties I will also give you some design tips on how to display your type.

@font-face allows you to use a font, even if it is not installed on the computer of the person browsing, by allowing you to specify a path to a copy of the font, which will be downloaded if it is not on the user's machine.

Because this technique allows a version of the font to be downloaded on a user's computer, it is important that the license for the font permits it to be used in this way. You add the font to your style sheet using the @font-face rule in a CSS sheet.

The book goes over a lot of stuff you can do with text, but I'm gonna gloss over the stuff that's already pretty obvious (e.g. bold, italics, etc.) and showcase the things which might actually be useful:

Leading (pronounced ledding) is the term typographers use for the vertical space inbetween lines. This might be useful for lists and such. This is done with the line-height attribute.

Kerning is the term typographers use for the space between each letter. You can control this with the letter-spacing attribute.

You can set text-alignment to 'justify', which indicates that every line in a paragrah, except the last line, should be set to take up the full width of the containing box.

You can use the text-indent property to indent the first line of text within an element. The amount you want to be indented can be indicated in em's or px.

You can use the :link and :visited (e.g. a:link { ... } etc.) in CSS files to set styles for links that have and have not been visited.




Responding to users:

  • :hover - This is applied when a user hovers over an element. Commonly used to change appearance of links, buttons, etc.

  • :active - This is applied when an element is activated by a user; for example, when a button is being pressed or a link being clicked. Sometimes this is used to make a button feel like it is actually being clicked.

  • :focus - This is applied when an element has focus. Any element you interact with, such as a link you can click on or any form control can have focus. Focus occurs when you browser discovers that you are ready to interact with an element on the page. For example, when your cursor is in a form and input ready to accept typing, that element is said to have focus. It is also possible to use the tab key on your keyboard to move through the interactive items on a page. When pseudo-classes are used, they should appear in this order: :link, :visited, :hover, :focus, :active.
